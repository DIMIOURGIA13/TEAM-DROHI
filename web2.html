Web 2.0 Overview

Web 2.0 (also known as participative (or participatory) web and social web) refers to websites that emphasize user-generated content, ease of use, participatory culture and interoperability (i.e., compatibility with other products, systems, and devices) for end users. The term was coined by Darcy DiNucci in 1999 and later popularized by Tim O'Reilly and Dale Dougherty at the first Web 2.0 Conference in 2004. Although the term mimics the numbering of software versions, it does not denote a formal change in the nature of the World Wide Web, but merely describes a general change that occurred during this period as interactive websites proliferated and came to overshadow the older, more static websites of the original Web.

Web 2.0 describes the current state of the internet, which has more user-generated content and usability for end-users compared to its earlier incarnation, Web 1.0. It does not refer to any specific technical upgrades to the internet; it refers to a shift in how the internet is used.

Comparison

FrontEnd Things

- HTML, CSS, and JavaScript work seamlessly together to create cohesive and interactive web experiences.
- HTML provides the structure, CSS styles the elements, and JavaScript adds behaviour and functionality.
- They interact through the Document Object Model (DOM), where JavaScript can access and manipulate HTML and CSS dynamically

BackEnd Things

- The backend is responsible for efficiently processing and storing vast amounts of user-generated data. This includes content uploads, interactions, and other dynamic elements, emphasizing a shift from static to dynamic web content. Server-Side Interactivity.
- The backend in Web 2.0 enables server-side interactivity, allowing for real-time updates and dynamic content delivery. This contrasts with Web 1.0, where websites were primarily static, and most processing occurred on the client side.
- Relies on the backend to provide APIs (Application Programming Interfaces) that facilitate seamless integration with third-party services and platforms. This promotes interoperability, enabling developers to create more interactive and feature-rich web applications.
